# `LegacyThunderIDNodeClient`

The `LegacyThunderIDNodeClient` class provides the full authentication API for server-side Node.js applications. The class implements OAuth 2.0 Authorization Code flow with PKCE, manages user sessions, and exposes methods for token retrieval, user information, and session lifecycle operations.

:::note
If you are building a framework-specific integration (such as an Express.js adapter), extend [`ThunderIDNodeClient`](/docs/next/sdks/node/apis/clients/thunderid-node-client) or subclass `LegacyThunderIDNodeClient` directly.
:::

## Usage

```ts title="src/auth.ts" showLineNumbers
import { LegacyThunderIDNodeClient } from '@thunderid/node'

const auth = new LegacyThunderIDNodeClient()

await auth.initialize({
  clientId: '<your-client-id>',
  serverOrigin: 'https://localhost:8090',
  afterSignInUrl: 'http://localhost:3000/callback',
  afterSignOutUrl: 'http://localhost:3000',
})
```

## Methods

### `initialize`

Initializes the client with the provided configuration and an optional custom storage backend. You must call `initialize` before using any other method.

```ts
initialize(config: AuthClientConfig<T>, store?: Storage): Promise<boolean>
```

#### Parameters

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `config` | `AuthClientConfig<T>` | ✅ | The client configuration object. See [`ThunderIDNodeConfig`](/docs/next/sdks/node/apis/config/thunderid-node-config) for available properties. |
| `store` | `Storage` | ❌ | Custom storage backend for session data. Defaults to an in-memory cache when not provided. |

#### Returns

`Promise<boolean>` — Resolves to `true` when initialization succeeds.

#### Example

```ts
await auth.initialize(
  {
    clientId: '<your-client-id>',
    serverOrigin: 'https://localhost:8090',
    afterSignInUrl: 'http://localhost:3000/callback',
    afterSignOutUrl: 'http://localhost:3000',
  },
  customStore,
)
```

### `signIn`

Initiates the sign-in flow for a user. On the first call (no authorization code present), the method invokes `authURLCallback` with the authorization URL and resolves with an empty token response. On the callback call (authorization code present), the method exchanges the code for tokens and resolves with the full `TokenResponse`.

```ts
signIn(
  authURLCallback: AuthURLCallback,
  userId: string,
  authorizationCode?: string,
  sessionState?: string,
  state?: string,
  signInConfig?: Record<string, string | boolean>,
): Promise<TokenResponse>
```

#### Parameters

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `authURLCallback` | `(url: string) => void` | ✅ | Callback that receives the authorization URL. Redirect the user to this URL to start the sign-in flow. |
| `userId` | `string` | ✅ | A unique identifier for the user session. Used to associate tokens with a specific user. |
| `authorizationCode` | `string` | ❌ | The authorization code returned by the identity provider after the user signs in. |
| `sessionState` | `string` | ❌ | The session state value returned by the identity provider. |
| `state` | `string` | ❌ | The state parameter returned in the redirect URL. |
| `signInConfig` | `Record<string, string \| boolean>` | ❌ | Additional parameters to include in the authorization request. |

#### Returns

`Promise<TokenResponse>` — Resolves with the token response after a successful token exchange, or with an empty token response when the authorization URL redirect is in progress.

#### Example

```ts
// Step 1 — Redirect user to authorization URL
app.get('/sign-in', (req, res) => {
  auth.signIn(
    (authUrl) => res.redirect(authUrl),
    req.cookies['session_id'],
  )
})

// Step 2 — Handle the callback
app.get('/callback', async (req, res) => {
  const { code, session_state, state } = req.query

  const tokens = await auth.signIn(
    (authUrl) => res.redirect(authUrl),
    req.cookies['session_id'],
    code as string,
    session_state as string,
    state as string,
  )

  res.redirect('/dashboard')
})
```

### `signOut`

Clears the user's session data and returns the identity provider sign-out URL. Redirect the user to this URL to complete the sign-out flow.

```ts
signOut(userId: string): Promise<string>
```

#### Parameters

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `userId` | `string` | ✅ | The unique identifier of the user session to end. |

#### Returns

`Promise<string>` — Resolves with the sign-out URL.

#### Example

```ts
app.get('/sign-out', async (req, res) => {
  const signOutUrl = await auth.signOut(req.cookies['session_id'])
  res.clearCookie('session_id')
  res.redirect(signOutUrl)
})
```

### `isSignedIn`

Checks whether the user has an active, valid session. The method also attempts a token refresh if the session has expired but a refresh token exists.


| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `userId` | `string` | ✅ | The unique identifier of the user session. |

#### Returns

`Promise<boolean>` — Resolves to `true` when the user has an active session.

#### Example

```ts
const signedIn = await auth.isSignedIn(req.cookies['session_id'])

if (!signedIn) {
  return res.redirect('/sign-in')
}
```

### `exchangeToken`

Exchanges the current token for a new token using a custom grant type. Use this method for advanced scenarios such as token exchange flows.

```ts
exchangeToken(config: TokenExchangeRequestConfig, userId?: string): Promise<TokenResponse | Response>
```

#### Parameters

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `config` | `TokenExchangeRequestConfig` | ✅ | Configuration object specifying the grant type, data, and request options. |
| `userId` | `string` | ❌ | The unique identifier of the user session. |

#### Returns

`Promise<TokenResponse | Response>` — Resolves with the token response or the raw server response, depending on the configuration.

#### Example

```ts
const tokenResponse = await auth.exchangeToken(
  {
    attachToken: false,
    data: {
      client_id: '<your-client-id>',
      grant_type: 'account_switch',
      scope: 'openid profile',
      token: currentAccessToken,
    },
    id: 'account-switch',
    returnResponse: true,
    returnsSession: true,
    signInRequired: true,
  },
  req.cookies['session_id'],
)
```
